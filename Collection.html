Collection

A Collection in Java is a framework that provides classes and interfaces to store and manipulate groups of objects.
The collections has three main branches: List, Set, Queue.
It makes handling data easier (adding, removing, searching, sorting).
It includes interfaces (like List, Set, Queue, Map) and their implementations (like ArrayList, HashSet, etc.).

collections is like a big container that holds groups of objects.

1]List

list Means Ordered collection (things stay in the order we put them).
It has Ordered collection means elements are stored in sequence.
it Allows duplicates.The same element can appear multiple times.
It has Index-based access . we can get elements by their position.
it Maintains insertion order so elements stay in the order you added them.

Common Implementations:

1)ArrayList

Based on dynamic arrays.
It is Fast for searching (random access).
Slower for inserting/removing in the middle.
ArrayList → like a resizable array.

2)LinkedList

Based on doubly linked list.
Fast for inserting/removing anywhere.
Slower for searching.
LinkedList → good for adding/removing items quickly.

3)Vector

Similar to ArrayList but synchronized (thread-safe).
Used less nowadays.
Vector → older version of ArrayList, synchronized.

2]Set

Set has Unique collection (no duplicates allowed).
Order depends on the type of Set.
No duplicates allowed . each element must be unique.
Order depends on implementation.
Useful when uniqueness matters (like storing IDs).

Common Implementations:

1)HashSet

In Hashset we Stores elements in a hash table.
Very fast for add, remove, search.
No guaranteed order.
HashSet → fast, but no order.

2)LinkedHashSet

Stores elements in hash table + linked list.
Maintains insertion order.

3)TreeSet

Stores elements in a tree structure.
Keeps elements sorted (ascending order).
Slower than HashSet but useful for sorted data.

3]Queue

Works like a line of people waiting.
It Follows FIFO (First In, First Out).
Used when processing tasks in order.
Some queues allow priority-based ordering.

Common Implementations:

1)PriorityQueue

Orders elements by priority (not just insertion order).
Example: tasks with higher priority executed first.

2)ArrayDeque

Double-ended queue.
Can add/remove from both ends.
Faster than LinkedList for queue operations.
LinkedList (also works as a Queue)
Can act as both List and Queue.
Examples: PriorityQueue → elements ordered by priority.
ArrayDeque → double-ended queue (can add/remove from both ends).

Map

Map (separate from Collection interface but part of framework)
Stores data as key-value pairs.
Keys must be unique, values can repeat.

Common Implementations:

1)HashMap

Stores data in hash table.
Very fast for search and insert.
No order maintained.

2)LinkedHashMap

Maintains insertion order of keys.

3)TreeMap

Stores keys in sorted order.
Slower than HashMap but useful for ordered maps.

4)Hashtable

Older version of HashMap.
Synchronized (thread-safe).
Rarely used now.

Examples: HashMap → fast, no order.
LinkedHashMap → keeps insertion order.
insertion order. TreeMap → keeps keys sorted.
Hashtable → older, synchronized.
