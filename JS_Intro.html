3]Variable

variable is like a container that stores data.
JavaScript gives us three keywords to declare variables

1]let

We Use let to declare a variable. We can change its value later. 

            it only works inside {} where it’s declared.
            Ex:
                let name = "Rutuja";
                name = "Jagtap"; // This is allowed

2]Var

It is also a way to declare variables. it Works like let, but it has different rules about where it's visible. 

            it works inside the whole function.
        
            Ex:
                var age = 25;
                age = 30;

3]Const

It is Used to declare a variable that cannot be changed. Once you give it a value, then we cannot change it. 

             Ex:
                const country = "India";
                country = "USA";
Points to Remember
Use let when the value will change.
Use const when the value should stay fixed.
Avoid var in modern code (it can cause confusing bugs).

5]Hoisting

What is hoisting:

Hoisting in JavaScript means that all variable and function declarations are moved to the top of their scope before the code runs.

During the creation phase, declarations (not initializations) are moved to the top of their scope. This is called hoisting

Ex: 
        Variables declared with var are hoisted and initialized with undefined
            console.log(x); // undefined
            var x = 5;
        let and const: These are hoisted too, but they remain in the Temporal Dead Zone (TDZ) until initialized. 
        Accessing them before declaration throws an error
            console.log(y); // ReferenceError
            let  y = 10;

Declarations are hoisted, initializations are not.
var are hoisted with undefined.
let and const are hoisted but inaccessible until initialized (TDZ).
Function declarations are fully hoisted.
