Join

In MySQL, a JOIN is used to combine rows from two or more tables based on a related column.
It helps you pull connected information together (like linking customers with their orders). There are several types of joins, each controlling how data is matched and displayed.

What is a JOIN ?
A JOIN links tables together using a common column (often a foreign key).
Example: We have a Customers table and an Orders table. To see which customer placed which order, you join them using CustomerID. Without joins, you’d have incomplete data because each table only stores part of the information.

Types of Joins in MySQL

1. INNER JOIN :
Returns rows where there is a match in both tables.
If a customer has placed an order, you’ll see them. If not, they won’t appear. its like "Show me only the overlaps."

2. LEFT JOIN (or LEFT OUTER JOIN) :
Returns all rows from the left table, and matched rows from the right table.
If there’s no match, you still see the left table’s data, but the right side will show NULL.
its like "Show me everything from the left, even if no match exists."

3. RIGHT JOIN (or RIGHT OUTER JOIN) :
Opposite of LEFT JOIN.
Returns all rows from the right table, and matched rows from the left table.
If no match, left side shows NULL.
Think of it as: "Show me everything from the right, even if no match exists."

4. FULL JOIN (or FULL OUTER JOIN) :
MySQL doesn’t support FULL JOIN directly, but you can simulate it using UNION.
Returns all rows when there is a match in either left or right table.
Think of it as: "Show me everything from both sides, matched or not."

5. CROSS JOIN :
Returns the Cartesian product (all possible combinations of rows).
If Customers has 10 rows and Orders has 5 rows, result = 50 rows.
Think of it as: "Mix everything with everything."

6. SELF JOIN:
A table joined with itself.
Useful for hierarchical data (like employees and their managers).
Think of it as: "Compare rows within the same table."
