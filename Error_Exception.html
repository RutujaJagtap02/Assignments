1.The Common Root:Throwable

In Java, both Error and Exception are subclasses of the Throwable class.
Only objects of type Throwable (or its subclasses) can be thrown using throw and caught using catch.

Error

What is an Error? :
Definition: Errors indicate serious problems that occur at runtime, usually outside the control of the program.
Errors = Fatal problems, outside program’s control.
Examples:
- OutOfMemoryError → JVM runs out of memory.
- StackOverflowError → Too many recursive calls overflow the stack.
- NoClassDefFoundError → Required class not found.
- Key Point: Errors are not meant to be caught or handled by applications. They usually signal that the program should terminate.
- Think of an Error like a system crash—we can’t fix it from inside our program.

Exception

What is an Exception?
Definition: Exceptions represent conditions that a program can anticipate and recover from.
In Java, exceptions can be caught and handled using try-catch blocks so the program doesn’t crash.
Exceptions = Handleable problems, inside program’s control.
Examples:
- IOException → Input/output operation fails.
- NullPointerException → Accessing a variable that points to null.
- ArithmeticException → Division by zero.
- Key Point: Exceptions should be caught and handled using try-catch blocks or declared with throws.

Types of Exceptions

1]Checked Exceptions:
- Checked at compile-time.
- Must be either handled (try-catch) or declared (throws).
- Example: IOException, SQLException.

2]Unchecked Exceptions (Runtime Exceptions):
- Checked at runtime.
- Not required to be handled explicitly.
- Example: NullPointerException, ArrayIndexOutOfBoundsException.
