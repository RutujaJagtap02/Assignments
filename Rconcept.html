1. Props (short for "Properties")

Props are like inputs you give to a component.
How they work: Props are read-only → you can’t change them inside the component.
They make components reusable because you can pass different values each time.

Example:
function Greeting(props) {
return Hello, {props.name}!;
}

Props = external data (like arguments to a function).

2. Hooks

Hooks are special functions in React that let you “hook into” features like state, lifecycle, and more.

Why they exist:
Before hooks, React used classes for state and lifecycle. Hooks made everything simpler and usable in functional components.

Common Hooks:
useState → lets you add state to a component.
useEffect → lets you run code when something changes (like fetching data or updating the page title).
useContext → lets you share data across components without passing props manually.
useRef → lets you directly access DOM elements or keep values between renders.

Hooks = special functions that give components superpowers (like state, side effects, etc.).
